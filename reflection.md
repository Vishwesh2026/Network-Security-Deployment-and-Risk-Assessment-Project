## Group reflection

I worked well when I planned tasks in tutorials and used quick messages to coordinate outside class. Zoom screen-sharing was especially useful for solving networking issues quickly.

Challenges:

- Scheduling time outside classes was sometimes difficult.
- I ran into a Git conflict early on when I edited the same file in parallel with other changes; I resolved it by stashing and pulling before merging.

What worked well:

- Regular GitHub commits tracked my progress.
- Weekly check-ins kept me on schedule.
- Screen-sharing sped up troubleshooting.

Improvements for next time:

- Set clearer internal deadlines for drafts.
- Share screenshots or outputs sooner to avoid blocking myself from making progress.

Suggested techniques for future projects:

- Use a task board (e.g., Trello) to visualise progress and due dates.
- Run short async standups via WhatsApp: "Yesterday / Today / Blockers".
- Aim for a small commit target each week so my contributions are visible.

These changes should reduce last-minute work and help me balance effort across the project.
