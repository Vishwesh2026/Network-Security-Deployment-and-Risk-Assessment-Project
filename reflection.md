## Group reflection

We worked well when we planned tasks in tutorials and used quick messages to coordinate outside class. Zoom screen-sharing was especially useful for solving networking issues quickly.

Challenges:
- Scheduling time outside classes was sometimes difficult.
- We ran into a Git conflict early on when both edited the same file; we resolved it by stashing and pulling before merging.

What worked well:
- Regular GitHub commits tracked progress.
- Weekly check-ins kept us on schedule.
- Screen-sharing sped up troubleshooting.

Improvements for next time:
- Set clearer internal deadlines for drafts.
- Share screenshots or outputs sooner to avoid blocking teammates.

Suggested techniques for future projects:
- Use a task board (e.g., Trello) to visualise progress and due dates.
- Run short async standups via WhatsApp: "Yesterday / Today / Blockers".
- Aim for a small commit target each week so contributions are visible.

These changes should reduce last-minute work and balance effort across the team.
