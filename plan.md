# Project Plan

**Network-Security-Deployment-and-Risk-Assessment-Project**

---

## Schedule

The table below shows the planned tasks and ownership by week. Leads completed tasks and the other member reviewed and contributed to the write-up.

| Week            | Tasks Planned                                                                                                             |
| --------------- | ------------------------------------------------------------------------------------------------------------------------- |
| Week 5          | Form the group, create the GitHub repository, agree on communication methods, draft this project plan                     |
| Week 6          | Set up VirtualBox, download and configure the OpenWRT VM, verify network connectivity                                     |
| Week 7          | Configure the web server on OpenWRT, document interfaces/IPs, draw the lab network diagram                                |
| Week 8          | Implement and test four firewall rules, draw the production network diagram                                               |
| Week 9          | Perform hardening steps: change default credentials, examine password storage, add SSH keys, disable unnecessary services |
| Week 10         | Capture and analyse HTTP/SSH traffic with `tcpdump` and Wireshark; export `.pcap` files and document findings             |
| Week 11         | Complete TVA Matrix risk assessment spreadsheet and identify the highest-risk data asset                                  |
| Week 12         | Recommend three NIST SP 800-53 security controls and finalise report sections                                             |
| Week 13         | Write the reflection, review all Markdown files, and record the demonstration video                                       |
| Week 13 (final) | Generate PDFs of the Markdown files, create the ZIP of the repo, and submit on Moodle; upload video to Echo360            |

---

Note: this schedule is a living plan. Tasks sometimes overlapped and the reflection documents any changes and reasons.
