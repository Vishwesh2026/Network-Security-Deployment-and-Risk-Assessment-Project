## Project checklist

Key project items (see the Project Specification for full requirements).

### Group and planning

- Form the group
- Create the GitHub repository via GitHub Classroom
- Agree on a communication plan (methods and frequency)
- Prepare the schedule in `plan.md`
- Make sure all members understand GitHub contribution expectations

### Network setup

- List assumptions
- Document the OpenWRT/VirtualBox network
- Draw the lab network diagram
- Host the test website on OpenWRT
- Configure the firewall rules
- Draw the production network diagram and assign IPs

### Security hardening and traffic analysis

- Change default credentials and inspect password storage
- Set up SSH key-based authentication
- Disable unnecessary services
- Capture and analyse HTTP traffic
- Capture and analyse SSH traffic

### Risk assessment and controls

- Conduct a mini risk assessment
- Recommend security controls
- Explain how the controls reduce risk

### Project reflection

- Include a screenshot of commits
- Document task split and compare with commit history
- Reflect on teamwork

### Report and files

- Report written in the project Markdown files on GitHub
- Screenshots are in the `images/` folder and linked from the report
- Packet captures are in the `captures/` folder and linked from the report
- Risk assessment spreadsheet: `risk-assessment.xlsx`
- Video demonstrating required tasks (when available)

### Submission checklist

- PDF of all Markdown files
- ZIP of the GitHub repository
- MP4 of the demonstration video
- Submit PDF and ZIP to Moodle
- Upload MP4 to the Echo360 link
